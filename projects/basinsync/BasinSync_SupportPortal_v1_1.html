Validation Rules Reference
All 9 Function 1 validation rules · Logic, thresholds, and pass/fail conditions · Internal reference only
🛡 Internal Reference Only — Vendors see only pass/fail outcomes and error messages, not the underlying rule logic or thresholds. Do not share this page or its contents externally.
01
Schema Integrity
BLOCK
18 mandatory columns present · correct data types · no blank mandatory cells · file format .xlsx confirmed
📋All 18 mandatory columns (A–V) must be present in exact order
📋Column E (Capex/Opex): all values must be in [CAP, OPX], no nulls
📋Column M (This Claim %): numeric, range 0.00–1.00
📋File extension must be .xlsx · MIME: application/vnd.openxmlformats
📋CLAIM_HEADER tab must be present · Vendor Declaration = TRUE
02
WBS Verification
BLOCK
Every WBS code in Column B must exist in SAP as an active Level 4 WBS element under the vendor's contracted project
📋Live SAP API call per unique WBS code in submission
📋WBS must be Level 4 work package — Level 3 or above = BLOCK
📋vendorId in Azure AD must match contract record for the project WBS
📋Project status must not be Closed or Suspended · billing block must not be set
03
Vendor ABN — Primary
BLOCK
Primary vendor ABN validated live against ATO ABR API · entity name cross-matched with Azure AD vendorCompany attribute
📋ABN format: 11 digits · passes ABN algorithm checksum (modulus 89)
📋ATO ABR API: entity must be Active and GST Registered
📋Entity name from ABR must match Azure AD vendorCompany field exactly (case-insensitive)
04
Sub-contractor ABN
WARN
All sub-contractor ABNs in Column R checked via ATO ABR API · new entities flagged for PM awareness
📋Same ABN checks as Rule 03 applied to each sub-contractor
📋MONITOR FLAG if entity registration age < 6 months
📋Not a BLOCK — valid new entities are permitted. PM sees advisory note.
05
GST Calculations
BLOCK
Column P = Column O × 0.10 · Column Q = Column O + Column P · exempt category check
📋Tolerance: ±$0.01 per row (rounding allowance)
📋GST-exempt categories checked against master exempt list
📋Any GST mismatch beyond tolerance = BLOCK with row/column reference returned
06
Cumulative % Cap
BLOCK
Prev Claimed % + This Claim % must not exceed 100% for any WBS line item
📋Col N (Prev Claimed %) + Col M (This Claim %) ≤ 1.0000 per row
📋Lookback: previous claims for same WBS fetched from claim_lines table
📋Over-claim = BLOCK with specific row and calculated overage returned
07
Rate Anomaly Detection
WARN
Unit rates in Column J compared against contracted schedule of rates + ±15% tolerance band
📋Contract rate fetched from SAP contract record for each WBS/cost element pair
📋MONITOR FLAG if submitted rate deviates >15% from contracted rate
📋Not a BLOCK — rate changes may be legitimately approved. PM sees flag + deviation %.
08
Duplicate Submission
BLOCK
Same vendor + project + claim period cannot be submitted twice · deduplication by hash
📋Hash of vendorId + projectWBS + claimPeriod checked against claim_headers
📋Existing claim in any non-rejected status = BLOCK
📋Vendor is shown the existing claim ID and its current status
09
SOPA Window Check
BLOCK
Claim submission must fall within the valid SOPA submission window for the reference period
📋Submission window: 1st–last business day of the month following the claim reference month
📋Early submission: BLOCK with next window open date returned
📋Late submission: WARN only — SOPA does not prevent late claims, but flags for PM awareness
